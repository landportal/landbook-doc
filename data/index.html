This document describes the RDF Data Portal developed as part of the LandBook project. The LandBook project is part of LandPortal

Initial version

Introduction

The LandBook project contains statistical data on land governance across the world.

The main sources of data are observations which have several dimensions, like the area (country or region), the time of the observation and the value.

The data portal follows the RDF Data Cube vocabulary where there are single observations with different dimensions grouped by slices and datasets.

Structural elements

Observations

A single observation. Conceptually, an observation is the most basic unit of information. It is usually a cell value from an excel sheet although observations can be obtained from other sources.

We consider three main dimensions for each observation:

  • cex:ref-area: A geographic area (a country or a region)
  • cex:ref-time: The time that is referred by that observation (usually a year or time interval)
  • cex:indicator:: The reference indicator

An observation has a numeric value cex:value

URI Template

{base}/observation/{observationId}
where 
{base} = Base URI for LandPortal 
{Dataset} = Dataset from which this observation has been obtained
{observationId} = Id of this observation

Schema

<Observation> { 
  a                      ( qb:Observation )
, cex:ref-area           @<Area>            # It can be a Country or a Region
, cex:ref-indicator      @<Indicator>       # Indicator
, cex:ref-time           @<Time>            # Time of observation: year, interval, etc.
, cex:value              xsd:double?        # Value of the observation (if obsStatus is not missing)
, cex:computation        @<Computation>     # Computation from which this observation has been obtained
, dcterms:issued         xsd:dateTime       # timestamp 
, qb:dataSet             @<DataSet>         # Dataset to which an observation belongs
, qb:slice               @<Slice>           # Slice to which this observation belongs
, rdfs:label             xsd:string         # Pattern "Value of {Region} in {Time} for indicator {Indicator}"@en
, sdmx-concept:obsStatus (                  # Observation status codes obtained from: http://purl.org/linked-data/sdmx/2009/code
                         sdmx-code:obsStatus-A  # Normal
                         sdmx-code:obsStatus-B  # Break
                         sdmx-code:obsStatus-E  # Estimated
                         sdmx-code:obsStatus-E  # Forecast
                         sdmx-code:obsStatus-I  # Imputed
                         sdmx-code:obsStatus-M  # Missing
                         sdmx-code:obsStatus-P  # Provisional
                         sdmx-code:obsStatus-S  # Strike
                         )
, lb:source              @<Upload>
}

Example

:o1 a qb:Observation 
 ; cex:ref-area           :France 
 ; cex:ref-time           :year2013 
 ; cex:ref-indicator      :hdi 
 ; cex:value              "2.3"^^xsd:double 
 ; cex:computation        cex:Raw 
 ; dcterms:issued         "2013-03-02T09:00:00"^^xsd:dateTime 
 ; qb:dataSet             :dataSet1 
 ; qb:slice               :slice1 
 ; rdfs:label             "Observation of Spain in 2013 for indicator hdi"@en 
 ; sdmx-concept:obsStatus sdmx-code:obsStatus-A
 ; lb:source              :upload1
 .

Indicators

Indicators are entities that provide specific information on the value of something.

For example, the Human development index is an indicator that provides information about the human development of a country.

Indicators can be simple or compound

URI Template

{base}/indicator/{indicatorId}
where 
{base} = Base URI for Landbook
{indicatorId} = Id of this indicator

Schema

<Indicator> {
  a   ( cex:Indicator )
, lb:preferable_tendency ( cex:decrease cex:increase cex:irrelevant )
, lb:measurement         @<Measurement>
, lb:last_update         xsd:dateTime
, lb:starred             xsd:boolean
, lb:topic               @<Topic>
, lb:indicatorType       shex:IRI  
, rdfs:label             xsd:string
, rdfs:comment           xsd:string
}

Example

:hdi a cex:Indicator 
 ; lb:preferable_tendency cex:decrease
 ; lb:measurement :measurement1
 ; lb:last_update "2014-04-04T11:50:18"^^xsd:dateTime
 ; lb:starred "true"^^xsd:boolean
 ; lb:topic :development
 ; lb:indicatorType lb:Simple
 ; rdfs:label "Human Development Index"@en 
 ; rdfs:comment "Longer description of indicator Human Development Index"@en
 .

Slice

A Slice is a grouping of observations.

URI Template

{base}/slice/{sliceId}
where 
{base} = Base URI for LandBook
{sliceId} = Id of this slice

Schema

<Slice> {
  a ( qb:Slice)
, cex:indicator @<Indicator>       # Indicator of this slice
, lb:dimension  (                  # Different types of dimensions in landbook. Can be:
                  lb:Time              # time varies
                  lb:Area              # area varies
                )?
, qb:observation @<Observation>*   # List of observations of this slice
}

Example

:slice1 a qb:Slice 
 ; cex:indicator :hdi 
 ; qb:observation :o1, :o2
 ; lb:dimension lb:Area  
 ; qb:dataSet :dataSet1 
 .

DataSet

A Dataset is a grouping of slices.

This concept is the same as the qb:DataSet concept from RDF Data Cube

URI Template

{base}/dataSet/{dataSetId}
where 
{base} = Base URI for Landbook
{dataSetId} = Id of this dataSet

Schema

<DataSet>{
  a ( qb:DataSet )
, lb:dataSource @<DataSource>
, lb:license @<License>
, qb:slice @<Slice>*                  # List of slices of this dataset
, sdmx-concept:freq (                 # Frequency. Codes from: http://purl.org/linked-data/sdmx/2009/code.
                                      # It can be:
                     sdmx-code:freq-A #   Annual
                     sdmx-code:freq-S #   Semester
                     sdmx-code:freq-Q #   Quarterly
                     sdmx-code:freq-M #   Monthly
                     sdmx-code:freq-W #   Weekly
                     sdmx-code:freq-D #   Daily
                     sdmx-code:freq-B #   Daily (business day)
                     sdmx-code:freq-N #   Minutely
                    )  
}

Example

:dataSet1 a qb:DataSet 
 ; sdmx-concept:freq sdmx-code:freq-A
 ; lb:dataSource :dataSource1 
 ; lb:license :license1
 ; qb:slice :slice1
 .

Countries, regions and areas

Countries and regions represent geographical areas.

Area represents the common parts of both countries and regions:

Schema

<Area> {
   a ( cex:Area )
 , rdfs:label xsd:string
}

Countries

Countries are represented by three main properties: The iso-codes with 2 and 3 letters and their FAO URI

Schema

<Country> &<Area> {
   lb:faoURI shex:IRI  
 , lb:iso2 xsd:string 
 , lb:iso3 xsd:string
 , lb:is_part_of @<Region>?
 , a (lb:Country)
}

URI Template

{base}/country/{countryCode}
where 
{base} = Base URI for LandBook 
{countryCode} = Iso-Code Alpha 3

Example

:France a cex:Area
 ; rdfs:label "France"@en
 ; lb:faoURI <http://www.fao.org/countryprofiles/index/en/?iso3=fra>
 ; lb:iso2 "FR"
 ; lb:iso3 "FRA" 
 ; lb:is_part_of :Europe
 .

Region

Concept of a region.

A region can be part of another region

URI Template

{base}/region/{regionId}
where 
{base} = Base URI for LandBook
{regionId} = Region Id

Schema

<Region> &<Area> {
   lb:UNCode xsd:string  
 , lb:is_part_of @<Region>?
}

Example

:Europe a cex:Area
 ; rdfs:label "Europe"@en
 ; lb:UNCode "EU"
 .

Time

Represents some time. There are several types of values to represent time values like years, time intervals or instants.

URI Template

{base}/organization/{orgId}
where 
{base} = Base URI for LandBook
{orgId} = Organization Id

Schema

<Time> {
  a ( cex:Time )
}

<Year> &<Time> {
 time:year xsd:gYear 
}

<Interval> {
    & <Time> ,
	time:hasBeginning xsd:dateTime,
	time:hasEnd       xsd:dateTime
}

Example

:year2013 a cex:Year 
 ; time:year "2013"^^xsd:gYear .

Organization

Organization

URI Template

{base}/organization/{orgId}
where 
{base} = Base URI for LandBook
{orgId} = Organization Id

Schema

<Organization> {
  a ( org:Organization )
, rdfs:label xsd:string
, foaf:homepage shex:IRI
, org:subOrganizationOf @<Organization>*
, org:hasSubOrganization @<Organization>*
}

Example

:faostat a org:Organization 
 ; rdfs:label "The statistics division of FAO"@en 
 ; foaf:homepage <http://faostat.fao.org/>
 .

Data source

A source of data

URI Template

{base}/datasource/{datasourceId}
where 
{base} = Base URI for LandBook
{datasourceId} = Datasource Id

Schema

<DataSource> {
  a (lb:DataSource)
, rdfs:label xsd:string
, lb:organization @<Organization>
, dcterms:creator @<User>
, dcterms:description xsd:string?
, dcterms:title xsd:string?
}

Example

:dataSource1 a lb:DataSource 
 ; rdfs:label "Data source name"@en 
 ; lb:organization :faostat
 ; dcterms:creator :john
 .

User

A user or agent that operates the system. It can be a person who uploads data to the system.

URI Template

{base}/user/{userId}
where 
{base} = Base URI for LandBook
{userId} = User Id

Schema

<User> {
  a (                             # Type of users (Agent's or People)
       foaf:Agent                 #    Agent could be some automatic web services 
       foaf:Person                #    Person    
    )                  
, rdfs:label xsd:string
, foaf:name xsd:string            # Name of user
, foaf:account xsd:string?        # Account of this user in the landportal
, org:memberOf @<Organization>+
}

Example

:john a foaf:Person
 ; rdfs:label "John Smith"
 ; foaf:name "John Smith"
 ; foaf:account "john"
 ; org:memberOf :faostat
 .

Upload

Represents an upload: a process by which some data has been uploaded into the system

URI Template

{base}/upload/{uploadId}
where 
{base} = Base URI for LandBook
{uploadId} = Upload Id

Schema

<Upload> {                        # Represents the process of uploading data into the system
  a              ( lb:Upload )    
, lb:user        @<User>          # User that uploaded the data
, lb:timestamp   xsd:dateTime           # Timestamp
, lb:ip          xsd:string             # IP address from which it has been uploaded
, lb:observation @<Observation>*  # List of observations that have been uploaded
, lb:dataSource  @<DataSource>    # Data source
}

Example

:upload1 a lb:Upload
 ; lb:user :john
 ; lb:timestamp "2014-04-04T11:50:18"^^xsd:dateTime
 ; lb:ip "156.34.56.23" 
 ; lb:observation :o1, :o2
 ; lb:dataSource :dataSource1
 .

Ontology

The Landportal book data portal terms are defined in the following ontologies:

  • Computex Ontology: Terms related to index definition and computation. Available in HTML and RDF/XML.
  • RDF Data Cube vocabulary: statistical terms. Described here and available in Turtle
